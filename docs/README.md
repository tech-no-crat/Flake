# Docs

Long-form notes that don't fit naturally in code comments.

- `how-tos/` — Reference guides and cheatsheets for day-to-day use.
- `upgrades/` — Dated journals of significant configuration changes (`YYYY-MM-DD-<topic>.md`).
- `incidents/` — Post-mortems for tricky bugs (`YYYY-MM-DD-<topic>.md`).

When adding a new entry, prefix the filename with the date so chronological
ordering is obvious in the file listing.
