## 1. System-wide Dark Mode

**Problem:** Apps (Firefox, Nautilus, pavucontrol, VLC, etc.) launched in
light mode with no consistent theming.

**Fix:** Configured a full dark-mode stack across all toolkits:

| Layer | Package / Setting |
|---|---|
| GTK3 | `adw-gtk3-dark` theme via `gtk.theme` |
| GTK4 / libadwaita | `dconf color-scheme = "prefer-dark"` |
| Qt5/Qt6 | `adwaita-dark` via `adwaita-qt` / `adwaita-qt6` |
| Icons | `Papirus-Dark` |
| Cursor | `Adwaita` 24px |
| Session env | `GTK_THEME`, `XCURSOR_THEME`, `XCURSOR_SIZE` |

## 2. Rofi "Cannot find theme" Error (Super+Shift+S)

**Problem:** `Super+Shift+S` (settings hub) and `Super+R` (app launcher)
both showed: `Error: Cannot find theme: nord`.

**Root cause:** `rofi-wayland` does not bundle third-party themes. The
`@theme "nord"` reference in `config.rasi` (generated by `programs.rofi.theme`)
had nothing to resolve to. The settings-hub script also passed `-theme nord`
as a CLI flag, compounding the error.

**Fix (two parts):**
1. Bundled a Nord-style `.rasi` file at `~/.local/share/rofi/themes/nord.rasi`
   via `home.file` â€” covers both the app launcher and the settings hub.
2. Removed the redundant `-theme nord` CLI flag from the settings-hub script
   (inherits the configured theme automatically).

Commits: `0c6f6c9`, `3a2ab78`

## 3. Home Manager Failing on Every Boot

**Problem:** `home-manager-shyam.service` failed at every boot with:
```
Existing file '/home/shyam/.config/gtk-3.0/settings.ini' would be clobbered
Existing file '/home/shyam/.config/gtk-4.0/settings.ini' would be clobbered
Existing file '/home/shyam/.config/gtk-4.0/gtk.css' would be clobbered
```

**Root cause:** GTK apps (e.g. `nwg-look`, GNOME apps) write regular files to
`~/.config/gtk-{3,4}.0/` at runtime. On the next boot, Home Manager tries to
replace them with symlinks but `backupFileExtension` creates `.hm-bak` files
that also accumulate and block subsequent runs.

**Fix:** Added a `home.activation.removeGtkConflicts` step that runs
`entryBefore ["checkLinkTargets"]` to delete the known conflicting files before
Home Manager's conflict check runs. Eliminates the failure permanently without
requiring `force = true` on individual files.

Commit: `5f1cca4`

## 4. Sunshine Showing No Apps on Moonlight (Primary Bug)

**Problem:** Moonlight connected successfully but showed a completely blank app
list â€” no Desktop, no Steam Big Picture, no games.

**Root cause (confirmed via Sunshine log):**
```
Error: No such node (env)
```
The `sunshine-gen-apps` script was writing `{"apps": [...]}` to `apps.json`.
Sunshine's JSON parser requires the root object to also contain an `"env"` key.
Without it, Sunshine silently discarded the entire app list on startup.

**Fix:** Changed the script to write `{"env": {}, "apps": [...]}`.

Commit: `095f370`

## 5. Sunshine Startup Race (Apps Empty on First Connection)

**Problem:** Even after the `env` fix, apps were missing if Moonlight connected
immediately after boot â€” before Hyprland's `exec-once` had run.

**Root cause:** `autoStart = true` caused Sunshine to start via systemd at boot
(before `exec-once` ran and generated `apps.json`). The app list was populated
only after `exec-once` restarted Sunshine, but Moonlight had already fetched the
empty list.

**Fix:** Set `autoStart = false`. Hyprland `exec-once` now:
1. Runs `sunshine-gen-apps` to write `apps.json`
2. Then runs `systemctl --user start sunshine`

Sunshine always has the full app list from the very first connection.

Commit: `276f31f`

## 6. Duplicate Games in Sunshine App List

**Problem:** Every game appeared twice in the Moonlight app list (e.g. two
"Counter-Strike 2", two "Slay the Spire 2", etc.). Desktop and Steam Big
Picture were unaffected.

**Root cause:** Steam libraries span multiple drives. `sunshine-gen-apps` scanned
all library roots and collected one `appmanifest_*.acf` entry per drive â€” so
any game installed or cached across two library paths was added twice.

**Fix:** Changed `recent_games()` to deduplicate by `appid` (keeping the entry
with the highest `LastPlayed` timestamp), so each game appears exactly once
regardless of how many library roots contain its manifest.
