# Modular NixOS Configuration

A flake-based NixOS setup for three hosts:

| Host                   | Hardware                                | Desktop  | Role                              |
| ---------------------- | --------------------------------------- | -------- | --------------------------------- |
| `nixos`                | Desktop, AMD Radeon RX 9070 XT          | Hyprland | Daily driver, gaming, Sunshine    |
| `surface-book-active`  | Surface Book 2, NVIDIA GPU (PRIME)      | GNOME    | Portable laptop with dGPU         |
| `surface-book-passive` | Surface Book 2, Intel iGPU only         | GNOME    | Light/remote-deployed laptop      |

Common settings live in `hosts/default/` and `home/default/`. Per-host configs import these and add only what differs. Optional features (gaming, GPUs, GNOME, Hyprland, audio variants, etc.) are reusable modules in `modules/`, composed per host inside `flake.nix`.

## Repository Layout

```

                              via the mkNixosConfig helper and selects which
                              modules each host gets.

  default/configuration.nix   Common system baseline: Nix settings, boot, graphics,

                              Imported by every host.

                              everything else comes from modules/surface-common
                              and modules/nvidia-surface.
  surface-book-passive/       Surface Book + Intel iGPU.
  <host>/hardware-configuration.nix   Generated by nixos-generate-config; per-host.

  default/home.nix            Common user packages and programs (git, neovim,
                              direnv, obsidian, nextcloud-client, firefox, …).
  nixos/home.nix              Imports default + Hyprland desktop setup, GTK theme
                              cleanup, desktop apps (discord, vscode, thunderbird).
  surface-book-active/home.nix    Imports default + moonlight-qt, nodejs, …
  surface-book-passive/home.nix   Imports default + lighter laptop set.

                              implicitly; flake.nix wires them per host.

  audio-laptop.nix            PipeWire with 32-bit ALSA (laptops).
  surface-common.nix          Timezone, locales, docker, common Surface packages.
  nvidia-surface.nix          NVIDIA driver + PRIME offload.
  intel-surface.nix           Intel iGPU only.
  gaming.nix                  Steam, Proton, gamemode, DXVK.
  multimedia.nix              GIMP, Blender, OBS, etc.
  sunshine.nix                Sunshine game-streaming host + helper scripts.
  docker.nix                  Docker daemon + shyam in docker group.
  1password.nix               1Password CLI + GUI + polkit integration.
  plex.nix                    Plex Media Server (currently unused; kept for reuse).
  nctalk.nix                  Nextcloud Talk desktop client (kept for reuse).

docs/                         Long-form notes that don't belong in code comments.
  how-tos/                    Reference guides (e.g. Hyprland keybindings).
  upgrades/                   Dated journals of significant config changes.
  incidents/                  Post-mortems for tricky bugs.

                              (currently the Samsung M2020 PPD).

                              that aren't yet wrapped as Nix derivations.
```

## How a Host Gets Built

`flake.nix` defines a small helper, `mkNixosConfig`, that wires up `nixosSystem`,
home-manager, and a list of modules. Each host entry only declares:

- `configPath`         — its own `hosts/<host>/configuration.nix`
- `homeConfigPath`     — its own `home/<host>/home.nix`
- `modules`            — extra NixOS modules to compose in
- `hardwareModules`    — modules from `nixos-hardware` (Surface devices)

Example (from `flake.nix`):

```nix
nixos = mkNixosConfig {
  configPath     = ./hosts/nixos/configuration.nix;
  homeConfigPath = ./home/nixos/home.nix;
  modules = [
    ./modules/hyprland.nix
    ./modules/audio.nix
    ./modules/sunshine.nix
    ./modules/gaming.nix
    ./modules/multimedia.nix
    ./modules/1password.nix
    ./modules/docker.nix
  ];
};
```

`hosts/<host>/configuration.nix` then imports `../default/configuration.nix` and
its own `hardware-configuration.nix`, and only declares what is unique to that
machine (hostname, GPU bits, filesystems, etc.).

Home Manager works the same way: `home/<host>/home.nix` imports
`../default/home.nix` and extends `home.packages` with `lib.mkAfter`.

## Build & Apply

```bash
# Validate the flake (all hosts)
nix flake check

# Build without switching
sudo nixos-rebuild build  --flake .#<hostname>

# Apply
sudo nixos-rebuild switch --flake .#<hostname>

# Build on a beefy box, deploy to a slow one
nixos-rebuild switch --flake .#surface-book-passive \
  --target-host shyam@surface-book-passive \
  --build-host  localhost \
  --use-remote-sudo
```

`<hostname>` is one of `nixos`, `surface-book-active`, `surface-book-passive`.

## Adding a New Host

1. Generate hardware config on the target machine:
   ```bash
   sudo nixos-generate-config --show-hardware-config > hosts/<host>/hardware-configuration.nix
   ```
2. Create `hosts/<host>/configuration.nix`:
   ```nix
   { config, pkgs, ... }:
   {
     imports = [
       ./hardware-configuration.nix
       ../default/configuration.nix
     ];
     networking.hostName = "<host>";
     system.stateVersion = "25.05";
     # …host-specific bits only
   }
   ```
3. Create `home/<host>/home.nix`:
   ```nix
   { config, pkgs, pkgs-unstable, lib, ... }:
   {
     imports = [ ../default/home.nix ];
     home.packages = lib.mkAfter (with pkgs; [ /* extras */ ]);
   }
   ```
4. Wire it up in `flake.nix`:
   ```nix
   <host> = mkNixosConfig {
     configPath     = ./hosts/<host>/configuration.nix;
     homeConfigPath = ./home/<host>/home.nix;
     modules        = [ ./modules/gnome.nix ./modules/audio.nix /* … */ ];
   };
   ```
5. `sudo nixos-rebuild switch --flake .#<host>`.

## Adding a New Module

Drop a file into `modules/`:

```nix
# modules/my-feature.nix
{ config, pkgs, lib, ... }:
{
  environment.systemPackages = [ pkgs.my-tool ];
  services.my-service.enable = true;
}
```

Then add `./modules/my-feature.nix` to the relevant host's `modules = [ … ]`
list in `flake.nix`. Modules are not auto-discovered — composition is explicit.

For toggleable modules, expose an option:

```nix
options.custom.myFeature.enable = lib.mkEnableOption "my feature";
config = lib.mkIf config.custom.myFeature.enable { … };
```

## Conventions

- **Common goes in `default/`**, host-specific stays in `hosts/<host>/` or
  `home/<host>/`. If a setting is needed by ≥2 hosts, hoist it.
- **Modules are opt-in.** Don't import them from `hosts/default/`; wire them in
  `flake.nix` so each host's feature set is visible in one place.
- **Use `lib.mkAfter` for `home.packages`** in per-host home configs to extend
  rather than replace the default list.
- **`pkgs-unstable`** is plumbed through `specialArgs` / `extraSpecialArgs`. Use
  it for packages that need to track unstable (e.g. `discord`, `vscode`).
- **Generations:** `boot.loader.systemd-boot.configurationLimit = 10` keeps the
  boot menu short. The `nixos` host also defines a `gnome` specialisation as a
  boot-menu fallback if Hyprland breaks.

## Troubleshooting

| Symptom                                              | Likely cause / fix                                                                 |
| ---------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `error: attribute 'pkgs-unstable' missing`           | A module/home file expects it but wasn't passed. Check `specialArgs` in `flake.nix`. |
| `option … is not defined`                            | The defining module isn't listed in that host's `modules` in `flake.nix`.          |
| `path '/nix/store/…' does not exist` after rebuild   | Stale `result` symlink. Safe to delete (`rm result`); it's gitignored.             |
| Home Manager fails on `checkLinkTargets` for GTK     | See the `removeGtkConflicts` activation script in `home/nixos/home.nix`.           |
| Slow rebuild on Surface Book passive                 | Use `--build-host` to build elsewhere; see *Build & Apply* above.                  |
| Sunshine drops connections                           | See `docs/incidents/2026-04-26-sunshine-streaming-disconnects.md`.                 |
